## 1. 网络体系结构与分层模型

OSI 七层 vs TCP/IP 四层的对照与映射关系。本节以体系结构图呈现各层的职责边界与服务访问点（SAP），阐述封装/解封的协议栈核心思想，分析分层带来的信息隐藏与模块化收益。

---

## 2. 物理层与数据链路层

物理层的信号编码与信道容量（奈奎斯特/香农）。数据链路层的成帧、差错检测（CRC）与介质访问控制（CSMA/CD、CSMA/CA）。本节以状态机描述以太网 CSMA/CD 的退避算法与 802.11 的 RTS/CTS 握手流程。

---

## 3. 网络层：IP 与路由

IPv4/IPv6 报文格式、子网划分与 CIDR。路由算法：距离向量（RIP）与链路状态（OSPF）的原理与收敛过程。本节以状态机描述路由表的更新过程，分析 ICMP 的差错报告机制，阐述 NAT 的地址转换体系结构。

---

## 4. 传输层：TCP 与 UDP

TCP 的三次握手与四次挥手的有限状态机（11 状态）。本节完整绘制 TCP 连接管理的状态转移图，分析滑动窗口协议的流量控制与拥塞控制（慢启动/拥塞避免/快重传/快恢复），对比 UDP 的无连接模型。

---

## 5. 应用层：HTTP

HTTP/1.0 → HTTP/1.1 → HTTP/2 → HTTP/3 的演进脉络。本节以状态机描述 HTTP 请求-响应的生命周期，分析 HTTP/2 的多路复用与帧格式，阐述 HTTP/3 基于 QUIC 的连接迁移与 0-RTT 握手。

---

## 6. 应用层：DNS

DNS 的分层命名体系与解析流程：递归查询 vs 迭代查询。本节绘制 DNS 解析的协议栈调用链，以状态机描述缓存命中/未命中的决策路径，分析 DNS 记录类型（A/AAAA/CNAME/MX/NS）与区域传送。

---

## 7. 网络安全

对称加密（AES）与非对称加密（RSA/ECC）的原理与体系结构。TLS 1.3 握手协议的状态机模型。本节以协议栈视角分析 IPSec 与 HTTPS 的加密层位置，阐述数字签名、证书链与 PKI 信任模型。

---

## 8. 协议栈数据封装与解封

自顶向下追踪一个 HTTP 请求的完整封装过程：应用数据 → TCP 段 → IP 数据报 → 以太网帧 → 物理信号。本节以逐层标注的体系结构图展示每层添加的协议首部，并反向描述接收端的解封与多路分解过程。
