## 1. 渗透测试标准

### 1.1 PTES 七阶段

| 阶段     | 说明           |
| -------- | -------------- |
| 前期交互 | 确定范围和目标 |
| 信息收集 | 被动/主动侦察  |
| 威胁建模 | 识别攻击路径   |
| 漏洞分析 | 发现安全缺陷   |
| 渗透攻击 | 利用漏洞       |
| 后渗透   | 深入内网       |
| 报告编写 | 交付成果       |

### 1.2 渗透测试类型

| 类型 | 测试者信息 | 授权     |
| ---- | ---------- | -------- |
| 黑盒 | 无信息     | 完全授权 |
| 白盒 | 完全信息   | 完全授权 |
| 灰盒 | 部分信息   | 完全授权 |

## 2. 信息收集

### 2.1 被动信息收集

```bash
# WHOIS
whois example.com

# DNS
dig example.com ANY
dnsrecon -d example.com

# 子域名
subfinder -d example.com
amass enum -d example.com

# 搜索引擎
site:example.com filetype:pdf
inurl:admin example.com

# 证书透明度
curl -s https://crt.sh/?q=%25.example.com
```

### 2.2 主动信息收集

```bash
# 端口扫描
nmap -sV -sC -p- target
nmap -sU --top-ports 100 target

# Web 指纹
whatweb https://target
nikto -h https://target

# 目录扫描
gobuster dir -u https://target -w wordlist.txt
ffuf -u https://target/FUZZ -w wordlist.txt
```

## 5. 报告编写

### 5.1 报告结构

1. 执行摘要
2. 范围和方法论
3. 漏洞详情（风险等级、复现步骤、修复建议）
4. 风险矩阵
5. 附录

### 5.2 风险评级

$$\text{风险} = \text{影响} \times \text{可能性}$$

| 等级 | CVSS范围 | 说明     |
| ---- | -------- | -------- |
| 严重 | 9.0~10.0 | 立即修复 |
| 高危 | 7.0~8.9  | 尽快修复 |
| 中危 | 4.0~6.9  | 计划修复 |
| 低危 | 0.1~3.9  | 酌情修复 |
