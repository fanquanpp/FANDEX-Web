## 1. 职业发展双通道

### 1.1 技术路线 vs 管理路线

| 维度     | 技术路线         | 管理路线           |
| :------- | :--------------- | :----------------- |
| 核心价值 | 技术深度和影响力 | 团队效能和业务成果 |
| 成长方向 | 技术专家         | 团队领导者         |
| 评价标准 | 技术贡献         | 团队产出           |
| 关键技能 | 架构、创新、攻坚 | 沟通、决策、培养   |
| 典型角色 | 架构师、技术专家 | 技术经理、总监     |

### 1.2 双通道模型

```
技术路线:                    管理路线:
初级工程师                   初级工程师
  │                           │
中级工程师                   中级工程师
  │                           │
高级工程师                   高级工程师
  │                           │
资深工程师 ──────→          技术组长
  │                           │
技术专家/架构师 ──→         技术经理
  │                           │
首席架构师 ──────→          技术总监
  │                           │
Fellow/CTO ──────→          VP/CTO
```

### 1.3 如何选择

| 适合技术路线     | 适合管理路线     |
| :--------------- | :--------------- |
| 享受解决技术难题 | 享受帮助他人成长 |
| 喜欢深度钻研     | 喜欢协调沟通     |
| 独立工作高效     | 团队协作高效     |
| 技术驱动         | 业务驱动         |

## 2. 职业规划框架

### 2.1 三环模型

```
      你擅长的
     ╱    ╲
    ╱  交集  ╲
   ╱ (甜蜜点) ╲
  ╱    ╱  ╲    ╲
你热爱的───社会需要的
```

### 2.2 IDP（个人发展计划）

| 步骤     | 说明                                  |
| :------- | :------------------------------------ |
| 自我评估 | 分析优势和不足                        |
| 设定目标 | 短期（1年）、中期（3年）、长期（5年） |
| 制定计划 | 具体的学习和发展行动                  |
| 执行跟踪 | 定期回顾和调整                        |

### 2.3 职业发展阶段

| 阶段   | 年限  | 核心任务                |
| :----- | :---- | :---------------------- |
| 学习期 | 0-3年 | 打好基础，快速学习      |
| 成长期 | 3-5年 | 建立专长，独立负责      |
| 突破期 | 5-8年 | 深度/广度突破，影响团队 |
| 成熟期 | 8年+  | 战略影响，行业贡献      |

## 3. 成长策略

### 3.1 技术成长

| 策略     | 说明              |
| :------- | :---------------- |
| T型人才  | 一个深度+多个广度 |
| 项目驱动 | 通过项目实践学习  |
| 开源参与 | 参与开源项目提升  |
| 技术分享 | 教是最好的学      |
| 持续学习 | 保持学习习惯      |

### 3.2 软技能成长

| 技能     | 说明             |
| :------- | :--------------- |
| 沟通表达 | 清晰表达技术方案 |
| 团队协作 | 跨团队合作能力   |
| 项目管理 | 驱动项目交付     |
| 影响力   | 无职权影响他人   |
