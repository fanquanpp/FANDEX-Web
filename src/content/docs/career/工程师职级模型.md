## 1. 职级体系概述

### 1.1 常见职级体系

| 体系   | 公司      | 特点                 |
| :----- | :-------- | :------------------- |
| P级    | 阿里      | P5~P14，数字越大越高 |
| T级    | 腾讯/百度 | T3~T12，技术序列     |
| Band级 | 华为      | 13~22级              |
| L级    | 字节      | 1-1~5-2              |
| E级    | Google    | L3~L10               |

### 1.2 职级对照（近似）

| 级别     | 阿里 | 腾讯 | 字节 | 角色              |
| :------- | :--- | :--- | :--- | :---------------- |
| 初级     | P5   | T4   | 1-2  | 初级工程师        |
| 中级     | P6   | T5   | 2-1  | 中级工程师        |
| 高级     | P7   | T6   | 2-2  | 高级工程师        |
| 资深     | P8   | T7   | 3-1  | 资深工程师/架构师 |
| 专家     | P9   | T8   | 3-2  | 技术专家          |
| 高级专家 | P10  | T9   | 4-1  | 首席架构师        |

## 2. 各级别能力要求

### 2.1 P5/P6 — 执行者

| 维度     | P5           | P6             |
| :------- | :----------- | :------------- |
| 技术深度 | 掌握基本技能 | 熟练掌握技术栈 |
| 独立性   | 需要指导     | 独立完成模块   |
| 影响范围 | 个人任务     | 单个模块       |
| 问题解决 | 按方案执行   | 能提出改进建议 |

### 2.2 P7 — 高级工程师

| 维度     | 要求           |
| :------- | :------------- |
| 技术深度 | 某领域专家     |
| 独立性   | 独立负责子系统 |
| 影响范围 | 跨模块影响     |
| 问题解决 | 定义问题并解决 |
| 团队贡献 | 指导初级工程师 |

### 2.3 P8 — 资深工程师/架构师

| 维度     | 要求             |
| :------- | :--------------- |
| 技术深度 | 多领域深度       |
| 独立性   | 主导技术方案     |
| 影响范围 | 跨团队影响       |
| 问题解决 | 解决复杂系统问题 |
| 团队贡献 | 培养高级工程师   |

### 2.4 P9+ — 技术专家/首席

| 维度     | 要求           |
| :------- | :------------- |
| 技术深度 | 行业级影响力   |
| 独立性   | 定义技术方向   |
| 影响范围 | 公司级/行业级  |
| 问题解决 | 解决战略级问题 |
| 团队贡献 | 培养架构师     |

## 3. 晋升核心要素

### 3.1 晋升公式

$$\text{晋升} = \text{已展现下一级能力} + \text{持续产出} + \text{影响力证据}$$

### 3.2 晋升准备

| 准备     | 说明                   |
| :------- | :--------------------- |
| 提前展现 | 晋升前已展现下一级能力 |
| 量化成果 | 用数据证明贡献         |
| 影响力   | 跨团队/跨部门影响      |
| 持续性   | 不是偶尔而是持续表现   |
