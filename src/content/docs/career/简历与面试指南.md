## 1. 简历撰写

### 1.1 简历结构

| 模块     | 内容                          | 篇幅        |
| :------- | :---------------------------- | :---------- |
| 基本信息 | 姓名、联系方式、GitHub/博客   | 2行         |
| 专业摘要 | 3-5年经验，专注领域，核心优势 | 3-4行       |
| 工作经历 | 公司、职位、时间、成果        | 每段3-5条   |
| 项目经验 | 项目名、角色、技术栈、成果    | 每项目3-4条 |
| 技术技能 | 按熟练度分类                  | 5-6行       |
| 教育背景 | 学校、专业、学位              | 1-2行       |

### 1.2 STAR法则

| 要素      | 说明 | 示例                           |
| :-------- | :--- | :----------------------------- |
| Situation | 背景 | 订单系统高峰期响应超时         |
| Task      | 任务 | 优化系统性能，目标P99<500ms    |
| Action    | 行动 | 引入Redis缓存、优化SQL、异步化 |
| Result    | 结果 | P99从2s降至300ms，QPS提升3倍   |

### 1.3 简历要点

| 要点       | 说明                          |
| :--------- | :---------------------------- |
| 量化成果   | 用数字说话（提升X%、降低Yms） |
| 突出亮点   | 最有价值的2-3个成果放前面     |
| 关键词匹配 | 根据JD调整关键词              |
| 一页原则   | 5年以下一页，10年以下两页     |
| 无错字     | 仔细检查拼写和语法            |

## 3. 面试技巧

### 3.1 算法面试

```
1. 理解题意 → 确认输入输出和约束
2. 思路分析 → 先说思路再编码
3. 编码实现 → 边写边解释
4. 测试验证 → 用示例和边界测试
5. 优化讨论 → 讨论时间和空间复杂度
```

### 3.2 系统设计面试

```
1. 需求澄清 → 问清功能和非功能需求
2. 容量估算 → 估算QPS、存储、带宽
3. 高层设计 → 画出系统架构图
4. 深入设计 → 详细设计核心组件
5. 权衡讨论 → 讨论方案的trade-off
```

### 3.3 面试心态

| 原则     | 说明                 |
| :------- | :------------------- |
| 沟通优先 | 思考过程比答案更重要 |
| 诚实为本 | 不会就说不会，不编造 |
| 保持冷静 | 面试是双向选择       |
| 提好问题 | 展示你的思考深度     |
