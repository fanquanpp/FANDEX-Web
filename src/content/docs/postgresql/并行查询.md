## 1. 并行查询概述

PostgreSQL 9.6+ 支持并行查询，利用多个 CPU 核心加速查询。

## 2. 并行查询类型

### 2.1 并行顺序扫描

```sql
-- 多个 worker 同时扫描表的不同部分
EXPLAIN ANALYZE SELECT COUNT(*) FROM large_table;
-- Gather
--   -> Parallel Seq Scan on large_table
--      Workers: 4
```

### 2.2 并行聚合

```sql
-- 每个 worker 部分聚合，然后汇总
EXPLAIN ANALYZE SELECT dept_id, AVG(salary) FROM employees GROUP BY dept_id;
-- Finalize Aggregate
--   -> Gather
--     -> Partial Aggregate
--       -> Parallel Seq Scan on employees
```

### 2.3 并行连接

```sql
-- 并行 Hash Join
EXPLAIN ANALYZE SELECT * FROM orders o JOIN customers c ON o.customer_id = c.id;
-- Gather
--   -> Parallel Hash Join
--     -> Parallel Seq Scan on orders
--     -> Parallel Hash
--       -> Parallel Seq Scan on customers
```

## 3. 配置

```sql
-- 最大并行 worker 数
ALTER SYSTEM SET max_parallel_workers_per_gather = 4;
ALTER SYSTEM SET max_parallel_workers = 8;
ALTER SYSTEM SET max_parallel_maintenance_workers = 4;

-- 触发并行扫描的最小表大小
ALTER SYSTEM SET min_parallel_table_scan_size = '8MB';

-- 代价阈值
ALTER SYSTEM SET parallel_tuple_cost = 0.1;
ALTER SYSTEM SET parallel_setup_cost = 1000;

-- 禁止并行查询
ALTER SYSTEM SET max_parallel_workers_per_gather = 0;
```

## 4. 监控

```sql
-- 查看并行查询执行情况
SELECT query, calls, total_exec_time,
       parallel_workers_to_launch,
       parallel_workers_launched
FROM pg_stat_statements
WHERE parallel_workers_launched > 0
ORDER BY total_exec_time DESC;
```
